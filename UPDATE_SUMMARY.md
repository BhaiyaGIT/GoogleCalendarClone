**After:** "A full-stack calendar app that works a lot like Google Calendar. Built with React on the frontend, Node.js + Express on the backend, and MongoDB for data storage."

### Example 2: Recurring Events

**Before:** "Recurring events are stored once with recurrence information. When querying a date range, instances are generated dynamically. Each instance gets a unique ID based on the original event and date."

**After:** "When you create an event that repeats (like 'every Monday'), I only store it once in the database with the recurrence info. When the frontend asks for events in a date range, the server automatically expands it into individual instances."

### Example 3: State Management

**Before:** "Zustand store for centralized state management with async action creators for API calls and state persistence across component re-renders."

**After:** "I'm using Zustand for state management - it's simpler and lighter than Redux. When you need to create an event, the store handles the API call and updates the calendar automatically."
